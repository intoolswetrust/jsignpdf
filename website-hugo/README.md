# JSignPdf website — Hugo + Hextra prototype

A parallel prototype of the JSignPdf website, built with
[Hugo](https://gohugo.io/) and the [Hextra](https://imfing.github.io/hextra/)
theme. The existing Docusaurus site under `../website/` is untouched; this
directory exists so both can be compared side-by-side.

## Why this exists

The current Docusaurus 2.4.1 stack pulls a deep React/MDX/Webpack dependency
tree that is hard to keep patched, and AsciiDoc is only supported through the
lossy `downdoc` shim. Hugo is a single Go binary with **no npm dependencies**,
handles AsciiDoc natively via the `asciidoctor` CLI, and Hextra delivers a
modern look out of the box.

## Single source of truth

The user guide lives in `../website/docs/JSignPdf.adoc`. That file is also
consumed by the Maven `asciidoctor-pdf` build (see `distribution/pom.xml`),
so it must stay where it is. Instead of duplicating it, `prepare.sh` copies
it into a Hugo page bundle at `content/docs/guide/index.adoc` together with
its `img/` assets. The destination paths are gitignored.

The site's static assets (logo, screenshots, social card) are copied the
same way from `../website/static/img/` into `static/img/`.

## Prerequisites

You only need **Docker**. The `hugomods/hugo:exts` image already contains
Hugo extended, Go (for fetching Hextra via Hugo modules), Node, Dart Sass and
Asciidoctor.

If you'd rather install the toolchain natively:

- Hugo **extended** ≥ 0.130
- Go ≥ 1.22
- Asciidoctor (`gem install asciidoctor` or `apt install asciidoctor`)

## Local development (Docker)

From the repo root:

```bash
# Production build → website-hugo/public/
docker run --rm -v "$PWD:/src" -w /src/website-hugo hugomods/hugo:exts \
  sh -c './prepare.sh && hugo --gc --minify'

# Live preview at http://localhost:1313/jsignpdf/
docker run --rm -it -p 1313:1313 -v "$PWD:/src" -w /src/website-hugo \
  hugomods/hugo:exts \
  sh -c './prepare.sh && hugo server --bind 0.0.0.0 --baseURL http://localhost:1313/jsignpdf/'
```

`prepare.sh` is idempotent — when running `hugo server`, edit
`../website/docs/JSignPdf.adoc`, re-run `prepare.sh` from another shell (or
just restart the container), and the browser will pick the change up.

## Local development (native toolchain)

```bash
cd website-hugo
hugo mod get -u github.com/imfing/hextra   # first time only
./prepare.sh
hugo server                                # http://localhost:1313/jsignpdf/
```

## Deployment

A GitHub Actions workflow lives at `../.github/workflows/hugo-deploy.yaml`.
It is **`workflow_dispatch`-only** for the prototype period so it doesn't
fight with the existing Docusaurus workflow (`doc-builder.yaml`) — both would
otherwise contend for the `github-pages` environment.

When you're ready to cut over:

1. Delete `../.github/workflows/doc-builder.yaml`.
2. Change the `on:` trigger in `hugo-deploy.yaml` to:
   ```yaml
   on:
     push:
       branches: [master]
       paths:
         - 'website-hugo/**'
         - 'website/docs/**'
         - '.github/workflows/hugo-deploy.yaml'
     workflow_dispatch:
   ```
3. Optionally rename `website-hugo/` → `website/` (after deleting the
   Docusaurus tree).

## Layout

```
website-hugo/
├── hugo.toml                    # site config + asciidoctor markup + menu
├── go.mod                       # Hugo module: imports github.com/imfing/hextra
├── prepare.sh                   # copies guide + static assets from ../website/
├── content/
│   ├── _index.md                # Hextra hero homepage
│   ├── docs/
│   │   ├── _index.md            # docs section landing (cascade type: docs)
│   │   └── guide/               # generated by prepare.sh — gitignored
│   │       ├── index.adoc       #   ← copy of website/docs/JSignPdf.adoc
│   │       └── img/             #   ← copy of website/docs/img/
│   └── screenshots/
│       └── _index.md            # gallery using Hextra cards
├── static/
│   └── img/                     # generated by prepare.sh — gitignored
└── .github/workflows/hugo-deploy.yaml   # at the repo root
```

## What you'll want to tweak

- **Hextra version** — `go.mod` currently pins `v0.9.7`. Run
  `hugo mod get -u github.com/imfing/hextra` periodically (or pin tighter).
- **`jsignpdf-version` attribute** — hard-coded in `hugo.toml` under
  `[markup.asciidocExt.attributes]`. Either bump it on each release or have
  the CI workflow inject it via `--config` / an env var, mirroring how the
  Maven build passes `${project.version}`.
- **Blog** — the prototype omits the blog. If you want it, add
  `content/blog/_index.md` with `cascade: { type: blog }` and Hextra will
  render it. Migrate the existing post from `../website/blog/` as a starting
  point.
- **Branding** — drop a custom CSS file under `assets/css/custom.css` and
  point Hextra at it via `params.customCSS` in `hugo.toml`.
