# Version 3.0.0

Major release with a brand-new desktop UI, modernized Windows packaging,
and new Linux distribution channels.

## Highlights

- **New JavaFX GUI** — a document-centric desktop experience with PDF
  preview, drag-to-place visible signatures, zoom/page navigation,
  drag-and-drop PDF loading, progress indication, and a collapsible
  side panel for signing options. The legacy Swing UI remains available
  as a fallback via `-Djsignpdf.swing=true`. Encrypted PDFs are now
  handled properly in the preview/signing flow.
- **New Windows installers built with `jpackage`** — replacing the old
  InnoSetup + Launch4j + Ant + Wine toolchain. Each release now ships
  three Windows x64 artifacts: an EXE installer, an MSI installer, and
  a portable ZIP with a bundled JRE. 32-bit Windows is no longer
  supported (Temurin no longer ships 32-bit JDKs); 32-bit and
  Linux/macOS users can still use the cross-arch `jsignpdf-<VERSION>.zip`
  with their own JRE.
- **Flatpak packaging** — JSignPdf is now published on Flathub as
  `io.github.intoolswetrust.JSignPdf`, with desktop integration and
  brand colors updated per the Flathub review.
- **Java 11 is now the minimum** runtime.

## GUI and UX

- Document-centric JavaFX workflow: open PDF → configure properties →
  place the visible signature rectangle on the page → sign.
- Signature Properties panel with hash algorithm, certification level,
  append mode, reason/location/contact fields.
- Signature Appearance panel (render mode, L2/L4 text, images, font),
  Timestamp/Validation and Encryption & Rights sections.
- Visible vs. invisible signature handling, encryption section
  visibility, encryption certificate field validation, and side panel
  hide/show fixes.
- "Reset Settings" action added to the JavaFX UI.
- New and updated translations (Czech, German, French, Spanish, Slovak,
  Portuguese, Croatian, Simplified Chinese, Traditional Chinese, and
  more — thanks to all Weblate contributors).

## Packaging and distribution

- `distribution/jpackage/` with shared JVM options and per-launcher
  configuration for `JSignPdf`, `JSignPdf-swing`, `JSignPdfC`
  (console), and `InstallCert`.
- `distribution/windows/build-windows-installers.ps1` drives the
  Windows job; the release workflow now runs a dedicated
  `windows-latest` job alongside the Ubuntu Maven deploy.
- MSI installers share a stable upgrade UUID so upgrades keep a single
  Add/Remove Programs entry.
- PDF user guide is bundled into the installed app under
  `<install>/app/docs/JSignPdf.pdf`.
- Flatpak manifests and metadata under `distribution/linux/flatpak/`.
- Per-version release notes files under
  `distribution/doc/release-notes/` are now attached as the GitHub
  Release body.

## Documentation and website

- Website migrated from the vulnerable Docusaurus stack to Hugo.
- User guide restructured around the new JavaFX UI, with a shared
  UI-agnostic "Signing options" reference chapter and a small
  Swing-for-backward-compatibility subsection.

## Other changes

- Improved symlink handling, including macOS (#168).
- Integration tests added for the PDF signing pipeline (#293).
- Moved publishing from OSSRH to Maven Central.
- Numerous dependency and GitHub Actions updates
  (commons-cli, commons-lang3, commons-io, maven-shade-plugin,
  maven-assembly-plugin, maven-gpg-plugin, jacoco, asciidoctor,
  actions/checkout, actions/setup-java, actions/setup-node,
  actions/upload-pages-artifact, actions/deploy-pages,
  github/codeql-action, svenstaro/upload-release-action, and others).

Thanks to everyone who contributed code, translations, bug reports,
and feedback!
